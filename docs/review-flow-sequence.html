Review Flow — UI to API Sequence

All HTTP calls the mobile app makes for the review feature  ·  SwapHaven API  ·  Jul 2026

Sealed-review policy: Reviews are invisible to both parties until (a) both submit, or (b) the 7-day window closes. The server enforces this — the client never receives sealed content. Only check the revealed flag from GET /review-status.

Endpoints at a glance

MethodPathAuthPurpose
POST /api/trades/:tradeId/complete Auth Mark trade done; opens 7-day review window
POST /api/trades/:tradeId/reviews Auth Submit sealed review (rating · comment · tags[])
GET /api/trades/:tradeId/review-status Auth Window state + revealed flag — re-poll on foreground
GET /api/trades/:tradeId/reviews/mine Auth Own submitted review (always readable by author)
GET /api/users/:userId/reviews Public Revealed reviews for profile display · cursor-paginated

Sequence diagrams

1 · Mark Trade Complete Screen: Chat / Trade Detail
2 · Submit a Review Screen: Write a Review
3 · Review Status (sealed → revealed) Screen: Review Status
4 · View My Submitted Review Screen: Review Status → tap 'View'
5 · Profile Reviews Tab Screen: User Profile → Reviews tab

Key payloads

POST /api/trades/:tradeId/reviews — request body

{
  "rating":  4,              // required · integer 1–5
  "comment": "string",       // optional · max 1000 chars
  "tags": [                  // optional · max 10 items
    "Fast reply",
    "Great condition",
    "Easy meetup",
    "Exactly as described",
    "Would trade again"
  ]
}

GET /api/trades/:tradeId/review-status — response

{
  "tradeId":        "uuid",
  "windowClosesAt": "2026-05-15T11:04:00.000Z",
  "windowOpen":     true,          // false after window closes
  "revealed":       false,         // true when both submitted OR window closed
  "myReview": {
    "submitted":   true,
    "submittedAt": "2026-05-08T11:12:00.000Z"
  },
  "theirReview": {
    "submitted": false             // no rating/comment until revealed=true
  }
}
